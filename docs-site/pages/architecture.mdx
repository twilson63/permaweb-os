# Architecture Overview

Permaweb OS is a Kubernetes platform where each user gets an isolated OpenCode pod with HTTPSig authentication.

## Mental Model

Think of each pod as a personal development server in the cloud:

1. **You don't SSH in** вҖ” You send signed JSON requests
2. **You don't manage it** вҖ” Kubernetes handles lifecycle
3. **You don't share it** вҖ” One wallet, one pod
4. **You can't exfiltrate from it** вҖ” Secrets are mounted, not accessible

## Components

### Gateway Service (`api/`)

A thin Express service that handles pod lifecycle:

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/api/auth/nonce` | Get challenge for wallet signature |
| `POST` | `/api/auth/verify` | Verify signature, get session token |
| `POST` | `/api/pods` | Create a new pod |
| `GET` | `/api/pods` | List pods for authenticated user |
| `GET` | `/api/pods/:id` | Get pod status |
| `DELETE` | `/api/pods/:id` | Delete a pod |

**The gateway never sees request payloads.** It only knows:
- Which wallet owns which pod
- Pod status (creating, running, stopped, error)
- Pod subdomain

### Frontend (`frontend/`)

React + Vite application that:
- Connects wallet (Arweave, MetaMask, etc.)
- Signs challenges with wallet
- Displays pod list
- Creates/deletes pods
- Streams JSONL responses

### HTTPSig Sidecar (`opencode-sidecar/`)

Node.js service (port 3001) that:
- Verifies HTTPSig signatures (RFC 9421)
- Supports Ethereum (`eth-personal-sign`) and Arweave (`arweave-rsa-pss-sha256`)
- Rejects unauthorized requests (401)
- Forwards valid requests to OpenCode
- Returns JSONL stream

### OpenCode Container (`images/opencode-base/`)

Docker image containing:
- OpenCode (agent harness)
- git, curl, wget
- brew (package manager)
- Node.js, Python, Rust, Go
- Standard dev tools

## Horizontal Pod Autoscaler

The API deployment uses HorizontalPodAutoscaler (HPA) for automatic scaling:

```yaml
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: api-hpa
  namespace: web-os
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: api
  minReplicas: 2
  maxReplicas: 10
  metrics:
    # CPU-based scaling
    - type: Resource
      resource:
        name: cpu
        target:
          type: Utilization
          averageUtilization: 70
    # Memory-based scaling
    - type: Resource
      resource:
        name: memory
        target:
          type: Utilization
          averageUtilization: 80
  behavior:
    scaleDown:
      stabilizationWindowSeconds: 300
      policies:
        - type: Percent
          value: 10
          periodSeconds: 60
        - type: Pods
          value: 2
          periodSeconds: 60
      selectPolicy: Min
    scaleUp:
      stabilizationWindowSeconds: 60
      policies:
        - type: Percent
          value: 100
          periodSeconds: 15
        - type: Pods
          value: 4
          periodSeconds: 15
      selectPolicy: Max
```

### HPA Thresholds

| Metric | Threshold | Behavior |
|--------|-----------|----------|
| CPU greater than 70% | Scale up | Add API pods (up to max) |
| Memory greater than 80% | Scale up | Add API pods (up to max) |
| Scale up stabilization | 60s | Prevent rapid fluctuation |
| Scale down stabilization | 300s (5m) | Prevent thrashing |

### Custom Metrics (Optional)

For latency-based scaling, enable Prometheus Adapter:

```yaml
# Requires prometheus-adapter
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: api-hpa-latency
  namespace: web-os
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: api
  minReplicas: 2
  maxReplicas: 10
  metrics:
    - type: Pods
      pods:
        metric:
          name: webos_http_request_duration_seconds
        target:
          type: AverageValue
          averageValue: 500m  # 500ms average response time
```

### Cost Analysis

### Recommended Node Types

| Tier | Node Type | Pods/Node | Monthly Cost | Cost/Pod |
|------|-----------|-----------|--------------|----------|
| Development | s-4vcpu-8gb | 8 | $48/mo | $6.00 |
| Production | c-4vcpu-8gb | 10 | $72/mo | $7.20 |
| High Performance | c-8vcpu-16gb | 20 | $144/mo | $7.20 |

### Scaling Triggers

| Signal | Action | Automation |
|--------|--------|------------|
| CPU greater than 70% sustained | Scale up API pods | HPA (automatic) |
| API pods at max (10) | Scale up nodes | Cluster Autoscaler |
| Node pool at max | Add more nodes | Manual + alert |
| Memory pressure greater than 90% | Upgrade node size | Manual + alert |
| Pending pods greater than 0 for 5m | Add nodes | Cluster Autoscaler |
| Less than 50% capacity for 30m | Scale down nodes | Cluster Autoscaler |
| Pod idle greater than 30m | Terminate pod | API logic |

For a detailed scaling strategy with vertical scaling, HA considerations, and cost optimization, see the [Scaling Strategy Document](/scaling-strategy).
