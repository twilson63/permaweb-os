# API Reference

Complete reference for all Permaweb OS API endpoints.

## Base URLs

| Environment | URL |
|-------------|-----|
| Production | `https://api.permaweb.live` |
| Staging | `https://api.staging.permaweb.live` |
| Local | `http://api.web-os.local:3000` |

## Authentication

All endpoints (except `/health`) require authentication via:

1. **Session Token** — `Authorization: Bearer <token>` (for pod lifecycle)
2. **HTTPSig Signature** — For direct pod requests

## Endpoints

### Authentication

#### Get Nonce

```http
POST /api/auth/nonce
Content-Type: application/json

{
  "address": "0xABC123..."
}
```

Returns a nonce for wallet signing.

**Request Body:**
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `address` | string | Yes | Wallet address (Ethereum `0x...` or Arweave 43 chars) |

**Response:**
```json
{
  "nonce": "nonce_abc123...",
  "expiresAt": "2026-03-11T11:00:00Z"
}
```

**Status Codes:**
- `200` — Success
- `400` — Invalid address format

---

#### Verify Signature

```http
POST /api/auth/verify
Content-Type: application/json

{
  "walletAddress": "0xABC123...",
  "signature": "0x...",
  "nonce": "nonce_abc123..."
}
```

Verifies wallet signature and returns session token.

**Request Body:**
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `walletAddress` | string | Yes | Wallet address |
| `signature` | string | Yes | Signed nonce (hex for Ethereum, base64url for Arweave) |
| `nonce` | string | Yes | Nonce from `/api/auth/nonce` |

**Supported Wallet Types:**
| Wallet Type | Address Format | Signature Format |
|------------|----------------|------------------|
| Ethereum | `0x` + 40 hex chars | `0x` + hex (personal_sign) |
| Arweave | Base64URL, 43 chars | Base64URL (RSA-PSS-SHA256) |

**Response:**
```json
{
  "sessionToken": "eyJhbGciOiJIUzI1NiIs...",
  "walletAddress": "0xABC123...",
  "walletType": "ethereum",
  "expiresAt": "2026-03-11T12:00:00Z"
}
```

**Status Codes:**
- `200` — Success
- `401` — Invalid signature
- `400` — Expired nonce

### Pod Management

#### Create Pod

```http
POST /api/pods
Authorization: Bearer <token>
Content-Type: application/json

{
  "model": "anthropic/claude-3-5-haiku"
}
```

Creates a new pod for the authenticated wallet.

**Parameters:**
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `model` | string | No | LLM model to use (default: `anthropic/claude-3-5-haiku`) |

**Response:**
```json
{
  "id": "abc123",
  "subdomain": "abc123.permaweb.live",
  "model": "anthropic/claude-3-5-haiku",
  "status": "creating",
  "createdAt": "2026-03-11T10:30:00Z"
}
```

**Status Codes:**
- `201` — Pod created
- `401` — Unauthorized
- `409` — Pod already exists for wallet
- `503` — Cluster at capacity

---

### LLM Key Management (Wallet-Scoped)

Each wallet has its own LLM API keys stored in isolated Kubernetes secrets. These endpoints allow wallet owners to manage their keys.

#### Register LLM Key

```http
POST /api/llm/keys
Authorization: Bearer <token>
Content-Type: application/json

{
  "provider": "openai",
  "apiKey": "sk-proj-abc123..."
}
```

Registers or updates an API key for a specific LLM provider.

**Request Body:**
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `provider` | string | Yes | LLM provider (`openai`, `anthropic`, `groq`) |
| `apiKey` | string | Yes | API key value (will be stored securely) |

**Response:**
```json
{
  "success": true,
  "provider": "openai",
  "createdAt": "2026-03-11T10:30:00Z"
}
```

**Status Codes:**
- `200` — Key registered successfully
- `401` — Unauthorized
- `400` — Invalid provider or key format

## Pod API (Direct)

Send requests directly to your pod's subdomain.

### Send Message

```http
POST https://{pod-id}.permaweb.live/verify
Signature: keyId="{wallet-address}", ...
Signature-Input: sig1=("@method" "@path" "host" "date")
Content-Type: application/json

{
  "id": "req_abc123",
  "method": "message",
  "params": {
    "content": "Create a hello.ts file"
  }
}
```

**Request Schema:**
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | string | Yes | Unique request ID |
| `method` | string | Yes | `"message"` |
| `params.content` | string | Yes | Message content |
| `params.sessionId` | string | No | Session ID for context |
| `timestamp` | number | No | Request timestamp |

**Response:** JSONL stream

See [JSONL Events](/jsonl-events) for event types.

**Status Codes:**
- `200` — Success (JSONL stream)
- `401` — Invalid signature
- `400` — Invalid request format

## Error Responses

All errors follow this format:

```json
{
  "error": {
    "code": "AUTH_INVALID_SIGNATURE",
    "message": "Signature verification failed",
    "details": {
      "reason": "Signature does not match keyId"
    }
  }
}
```

### Error Codes

| Code | HTTP Status | Description |
|------|-------------|-------------|
| `AUTH_INVALID_SIGNATURE` | 401 | Signature verification failed |
| `AUTH_EXPIRED_NONCE` | 400 | Nonce has expired |
| `AUTH_REPLAY_DETECTED` | 400 | Request was already processed |
| `POD_NOT_FOUND` | 404 | Pod does not exist |
| `POD_NOT_RUNNING` | 503 | Pod is not in running state |
| `POD_LIMIT_EXCEEDED` | 429 | Pod limit for wallet exceeded |
| `REQUEST_INVALID_FORMAT` | 400 | Invalid JSON format |
| `REQUEST_MISSING_FIELD` | 400 | Required field missing |
| `REQUEST_TOO_LARGE` | 413 | Request body exceeds size limit |
| `LLM_KEY_NOT_FOUND` | 500 | No LLM key configured |
| `LLM_RATE_LIMITED` | 429 | LLM API rate limit exceeded |
| `LLM_ERROR` | 500 | LLM provider returned error |
