# Security Model

Permaweb OS uses cryptographic signatures for authentication and multi-tenant isolation for security.

## Overview

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                            SECURITY BOUNDARIES                               │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│   ┌──────────────────┐                                                        │
│   │   User's Wallet   │  ← Identity lives here                               │
│   │   (private key)   │  ← Never shared, never leaves device                 │
│   └──────────────────┘                                                        │
│            │                                                                  │
│            │ Signs request                                                    │
│            ▼                                                                  │
│   ┌──────────────────┐                                                        │
│   │   HTTP Request    │  ← Signed payload                                     │
│   │   + Signature     │  ← Includes keyId (wallet address)                    │
│   └──────────────────┘                                                        │
│            │                                                                  │
│            ▼                                                                  │
│   ┌──────────────────┐                                                        │
│   │  Pod Subdomain    │  ← {pod-id}.permaweb.live                               │
│   │                   │  ← Direct access, no gateway                          │
│   └──────────────────┘                                                        │
│            │                                                                  │
│            ▼                                                                  │
│   ┌──────────────────┐                                                        │
│   │  HTTPSig Sidecar  │  ← Verifies signature                                 │
│   │                   │  ← Checks keyId == pod owner                           │
│   │                   │  ← Rejects if mismatch (401)                           │
│   └──────────────────┘                                                        │
│            │                                                                  │
│            │ Only if valid                                                     │
│            ▼                                                                  │
│   ┌──────────────────┐                                                        │
│   │    OpenCode       │  ← Processes request                                  │
│   │                   │  ← Reads /secrets/llm/ for API key                    │
│   │                   │  ← Never exposes secrets to user                      │
│   └──────────────────┘                                                        │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

## Key Design Decisions

### 1. Wallet = Identity

No usernames, no passwords. Your wallet is your identity.

```
Traditional: User → Password → Database (they see everything)
Permaweb OS: User → Wallet Signature → Your Pod
```

Both **Arweave** and **Ethereum** wallets are supported:

| Wallet Type | Address Format | Signature Algorithm |
|------------|----------------|---------------------|
| Arweave | Base64URL, 43 chars | RSA-PSS-SHA256 |
| Ethereum | `0x` + 40 hex chars | personal_sign (ECDSA) |

### 2. Per-Pod Isolation

Each wallet gets exactly one pod. Each pod is:

- **Isolated** — Container-level isolation
- **Non-root** — Runs as UID 1000
- **Resource-limited** — CPU/Memory quotas enforced
- **Network-isolated** — No pod-to-pod communication

### 3. HTTPSig for Authentication

We use HTTP Message Signatures (RFC 9421) because:

- **Asymmetric crypto** — No shared secrets
- **Standard** — RFC 9421 is an IETF standard
- **Flexible** — Works with RSA, ECDSA, Ethereum signatures, Arweave signatures
- **Verifiable** — Anyone can verify with the public key

### 4. Admin-Provided LLM Keys

Users don't bring their own API keys. The platform provides them:

```
Traditional AI Platform:
  User → Brings own API key → Stored in database → Risk of exposure

Permaweb OS:
  Admin → Sets cluster-wide LLM keys → Mounted to pods securely
                                              │
  User → Creates pod → Uses admin's keys (never sees them)
                                              │
  Platform → Tracks usage per wallet → Admin bills accordingly
```

**Benefits:**
- Users don't need LLM API keys to get started
- Admin manages and rotates keys centrally
- Keys are never exposed to users (mounted read-only)
- Usage is tracked per wallet for billing

## Multi-Tenant Security

Permaweb OS implements defense-in-depth for multi-tenant isolation. Each wallet's data and secrets are isolated from other wallets.

### Per-Wallet Secret Isolation

Each wallet gets its own Kubernetes secret for LLM API keys:

```
Wallet A (0xABC123...) → llm-keys-a1b2c3d4e5f67890 → Pod A
Wallet B (0xDEF456...) → llm-keys-f7e8d9c0b1a23456 → Pod B

Pod A CANNOT access Pod B's secrets
Pod B CANNOT access Pod A's secrets
```

**Implementation:**

1. **Secret Naming**: Secrets use a deterministic name based on wallet hash:
   ```
   secret-name = "llm-keys-" + SHA256(walletAddress)[:16]
   ```

2. **Pod Mounting**: Each pod mounts only its owner's secret:
   ```yaml
   volumes:
     - name: llm-secrets
       secret:
         secretName: llm-keys-a1b2c3d4e5f67890  # Wallet-specific
         
   volumeMounts:
     - name: llm-secrets
       mountPath: /secrets/llm
       readOnly: true
   ```

3. **RBAC Enforcement**: Kubernetes RBAC prevents pods from accessing other secrets:
   ```yaml
   # Pod can only read its own secret
   rules:
     - apiGroups: [""]
       resources: ["secrets"]
       resourceNames: ["llm-keys-a1b2c3d4e5f67890"]
       verbs: ["get"]
   ```

4. **NetworkPolicy**: Network policies prevent pod-to-pod communication:
   ```yaml
   # Pods cannot talk to each other
   apiVersion: networking.k8s.io/v1
   kind: NetworkPolicy
   spec:
     podSelector: {}
     policyTypes:
       - Ingress
       - Egress
     ingress:
       - from: []  # Only from ingress
     egress:
       - to: ["0.0.0.0/0"]  # Internet only
         - to: ["10.0.0.0/8"]  # Internal services (not other pods)
   ```

### Per-Wallet Owner Keys

Each wallet gets a unique RSA key pair for HTTPSig verification:

```
┌────────────────────────────────────────────────────────────────────────────┐
│                        OWNER KEY LIFECYCLE                                 │
├────────────────────────────────────────────────────────────────────────────┤
│                                                                            │
│  1. Wallet Registration                                                    │
│     └── API generates RSA-2048 key pair                                   │
│     └── Private key returned to wallet owner (never stored)              │
│     └── Public key stored in K8s Secret: owner-key-<keyId>               │
│                                                                            │
│  2. Pod Creation                                                          │
│     └── Secret mounted ONLY in sidecar container                          │
│     └── OpenCode container has NO access                                  │
│                                                                            │
│  3. Request Verification                                                  │
│     └── Sidecar reads public key from mounted secret                     │
│     └── Verifies signature matches owner wallet                          │
│     └── If valid → forwards to OpenCode                                  │
│     └── If invalid → 401 Unauthorized                                     │
│                                                                            │
└────────────────────────────────────────────────────────────────────────────┘
```

**Why Per-Wallet Keys?**

- **Key Compromise Isolation**: If one wallet's key is compromised, others remain secure
- **Key Rotation**: Individual wallets can rotate keys without affecting others
- **Audit Trail**: Each signature is tied to a specific wallet's key
- **No Shared Secrets**: No single master key that could compromise all pods

**Key Storage:**

```yaml
# Kubernetes Secret for owner public key
apiVersion: v1
kind: Secret
metadata:
  name: owner-key-a1b2c3d4
  namespace: web-os
  labels:
    app.kubernetes.io/name: owner-key
    web-os.io/wallet-hash: a1b2c3d4
type: Opaque
data:
  publicKey.pem: <base64-encoded-pem>
```

**Mount Configuration:**

```yaml
# Only sidecar gets owner key
containers:
  - name: sidecar
    volumeMounts:
      - name: owner-key
        mountPath: /secrets/owner
        readOnly: true
        
  - name: opencode
    # NO access to owner-key volume
    
volumes:
  - name: owner-key
    secret:
      secretName: owner-key-<keyId>
```

## What's Protected

| Asset | Location | Access |
|-------|----------|--------|
| Wallet private key | User's device | Only user |
| Owner key (per-wallet) | K8s Secret | Only sidecar container |
| LLM API keys | K8s Secret (per-wallet) | Only that wallet's OpenCode |
| Pod subdomain | DNS | Public (but requests must be signed) |
| Request signatures | In-flight | Sidecar validates, then discards |
| OpenCode state | Pod filesystem | Only that pod's OpenCode |

## What's NOT Protected

| Asset | Why |
|-------|-----|
| Pod subdomain | Public by design — anyone can see a pod exists |
| Pod count | Public — anyone can see how many pods exist |
| Request timing | Metadata — not critical |

## Secret Management

### LLM API Keys

Each wallet's LLM keys are stored in separate Kubernetes Secrets:

```yaml
# Wallet-specific LLM keys secret
apiVersion: v1
kind: Secret
metadata:
  name: llm-keys-a1b2c3d4e5f67890
  namespace: web-os
  labels:
    app.kubernetes.io/name: web-os-llm-keys
    web-os.io/wallet-hash: a1b2c3d4e5f67890
  annotations:
    web-os.io/owner-wallet: "0xabc123..."
type: Opaque
data:
  openai: <base64-encoded-key>
  anthropic: <base64-encoded-key>
```

OpenCode reads from:
- `/secrets/llm/openai` — OpenAI API key
- `/secrets/llm/anthropic` — Anthropic API key
- `/secrets/llm/groq` — Groq API key (optional)

**Users never see these values** — they're mounted into the pod, used by OpenCode, and never returned via API.

### Owner Verification Keys

Each wallet has a unique HTTPSig verification key:

```yaml
# Owner public key for HTTPSig verification
apiVersion: v1
kind: Secret
metadata:
  name: owner-key-<keyId>
  namespace: web-os
type: Opaque
data:
  publicKey.pem: <base64-encoded-pem>
```

**Security properties:**
- Private key is returned to the wallet owner ONCE at registration
- Platform never stores the private key
- Public key is mounted ONLY in the sidecar container
- OpenCode container has NO access to owner keys

## Request Signing

Every request to a pod must be signed:

### Ethereum Signing

```typescript
import { signatureHeaders } from 'http-message-sig';

const headers = await signatureHeaders({
  method: 'POST',
  url: '/verify',
  headers: {
    host: podSubdomain,
    date: new Date().toUTCString()
  }
}, {
  signer: {
    keyid: wallet.address.toLowerCase(),
    sign: async (data) => {
      const signature = await wallet.signMessage(data);
      return Buffer.from(signature.slice(2), 'hex');
    }
  }
});
```

### Arweave Signing

Arweave uses RSA-PSS-SHA256 signatures:

```typescript
import { signatureHeaders } from 'http-message-sig';

const headers = await signatureHeaders({
  method: 'POST',
  url: '/verify',
  headers: {
    host: podSubdomain,
    date: new Date().toUTCString()
  }
}, {
  signer: {
    keyid: walletAddress, // Arweave address (43 chars)
    alg: 'arweave-rsa-pss-sha256',
    sign: async (data) => {
      const signature = await window.arweaveWallet.signMessage(
        new TextEncoder().encode(data),
        { name: 'RSA-PSS', saltLength: 32 }
      );
      return new Uint8Array(signature);
    }
  }
});
```

## Replay Protection

The sidecar includes replay protection:

1. **Timestamp check** — Request must be within ±5 minutes
2. **Nonce check** — Each request must have a unique `id`
3. **Cache** — Recent request IDs are cached and rejected if reused

```typescript
// Request must include fresh timestamp and unique ID
const request = {
  id: `req_${Date.now()}_${randomUUID()}`,  // Unique ID
  timestamp: Date.now(),                      // Fresh timestamp
  method: "message",
  params: { content: "Hello" }
};
```

## Best Practices

### For Users

1. **Keep your private key secure** — Never share it
2. **Use a hardware wallet** — For production use
3. **Verify subdomain** — Always check you're sending to the correct pod
4. **Store your owner key** — Private key returned at registration is your responsibility

### For Cluster Admins

1. **Rotate LLM keys regularly** — Update the K8s Secrets
2. **Monitor usage** — Track token consumption per wallet
3. **Set resource limits** — Prevent runaway pods
4. **Enable audit logging** — Track all API calls
5. **Use NetworkPolicy** — Enforce pod isolation
6. **Separate secrets per wallet** — Never share LLM keys across wallets
