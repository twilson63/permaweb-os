# Welcome to Permaweb OS

A Kubernetes-based platform where each user connects with their wallet, spawns a personal OpenCode pod, and interacts through cryptographically signed requests. **Your pod, your code, your keys.**

## Why Permaweb OS?

| Traditional SaaS | Permaweb OS |
|-----------------|-------------|
| User account + password | Wallet = identity |
| Shared infrastructure | Your own pod |
| API keys in their database | Admin provides LLM keys (never exposed) |
| They see your conversations | Only you see your conversations |
| Trust them with your data | You control your data |

## The Flow

```
1. Connect Wallet     → Sign challenge with Arweave/Ethereum/RSA
2. Create Pod         → Spawn personal OpenCode container
3. Send Signed Request → HTTPSig-signed JSON to your pod subdomain
4. Stream Response    → JSONL events stream back to you
```

## Architecture at a Glance

```
┌─────────────────────────────────────────────────────────────┐
│                     permaweb.live                            │
├─────────────────────────────────────────────────────────────┤
│  api.permaweb.live  ←  Gateway (create/delete pods)         │
│                                                              │
│  abc123.permaweb.live  ←  Your Pod                          │
│  ├── HTTPSig Sidecar (verifies your signature)              │
│  └── OpenCode (processes requests, uses admin's LLM keys)   │
│                                                              │
│  xyz789.permaweb.live  ←  Another Pod                       │
│  └── ...                                                     │
└─────────────────────────────────────────────────────────────┘
```

## Key Concepts

- **Wallet Authentication** — Connect with Arweave, RSA, ECDSA, or Ethereum wallets
- **HTTPSig Verification** — RFC 9421 compliant request signing
- **Per-Pod Isolation** — Each user gets their own container
- **Admin-Provided LLM Keys** — Users don't need their own API keys
- **Subdomain Routing** — Each pod gets `{pod-id}.permaweb.live`
