🌐 Web OS Architecture

Kubernetes platform for personal OpenCode pods with HTTPSig authentication

web-os.live

Overview

Each user connects with their wallet (Arweave/RSA/ECDSA), spawns a personal OpenCode pod, and interacts via HTTPSig-signed JSON requests. Each pod runs a developer agent image with git, curl, brew, and standard development tools.

Architecture Decision Records

ADR-001: Wallet-Based Authentication

Accepted

Use Arweave/RSA/ECDSA wallets with HTTPSig for authentication. One wallet per pod.

ADR-002: JSON In → JSONL Out

Accepted

Input: JSON object. Output: JSONL stream. Simple, streamable, agent-friendly.

ADR-003: Developer Agent Image

Accepted

Base image includes OpenCode + git + curl + brew + dev tools. ~629MB.

ADR-004: Per-Pod HTTPSig Verification

Accepted

Each pod has HTTPSig sidecar that verifies all requests against owner wallet.

ADR-005: Secret Mount Pattern

Accepted

LLM keys stored in K8s Secrets, mounted as files at /secrets/llm/.

ADR-006: Per-Pod Subdomain

Accepted

Each pod gets {pod-id}.web-os.live subdomain for direct access.

Request Flow

1. User authenticates at api.web-os.live with wallet signature
2. Gateway creates pod with subdomain {pod-id}.web-os.live
3. User sends signed JSON to {pod-id}.web-os.live
4. HTTPSig sidecar verifies signature against owner wallet
5. OpenCode processes request, returns JSONL stream
6. Response streams back to user

Components

api/

Gateway API service

  • POST /api/pods - Create pod
  • GET /api/pods - List pods
  • GET /api/pods/:id - Pod status
  • DELETE /api/pods/:id - Delete pod

frontend/

Vite React frontend

  • Pod list with auto-refresh
  • Create/Delete buttons
  • Wallet connection (coming)

opencode-sidecar/

HTTPSig verification

  • Port 3001
  • Validates request signatures
  • Rejects unauthorized requests

images/opencode-base/

Developer agent image

  • OpenCode agent harness
  • git, curl, wget, brew
  • Node.js, Python, Rust, Go
