📖 Developer Guide
How to use the PermawebOS Agent API
// 1. Create a session const session = await fetch('http://localhost:4096/session', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ title: 'My Session' }) }).then(r => r.json()); // 2. Send a message (sync - waits for response) const response = await fetch(`http://localhost:4096/session/${session.id}/message`, { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ parts: [{ type: 'text', text: 'Hello!' }], model: 'anthropic/claude-opus-4-6' }) }).then(r => r.json()); // 3. Or send async and listen for events const es = new EventSource('http://localhost:4096/event'); es.onmessage = (e) => console.log(JSON.parse(e.data));

Session Endpoints

POST /session Body: { "title": "My Session", "parentID": "optional" } Response: { "id": "ses_xxx", "title": "My Session", ... } GET /session Response: [{ "id": "ses_xxx", ... }] GET /session/:id Response: { "id": "ses_xxx", "title": "...", ... } DELETE /session/:id Response: true

Message Endpoints

POST /session/:id/message (Sync - waits for response) Body: { "parts": [{ "type": "text", "text": "Hello!" }], "model": "anthropic/claude-opus-4-6", "agent": "optional" } Response: { "info": {...}, "parts": [...] } POST /session/:id/prompt_async (Async - no wait) Body: Same as /message Response: 204 No Content GET /session/:id/message Response: [{ "info": {...}, "parts": [...] }]

Server-Sent Events

GET /event Response: SSE stream Event Types: - server.connected - Initial connection - session.updated - Session state changed - message.updated - New message part - message.part.updated - Part streaming - file.edited - File was modified - command.executed - Command ran // Listen for events const es = new EventSource('/event'); es.addEventListener('message.updated', (e) => { const data = JSON.parse(e.data); console.log('Message:', data); });
